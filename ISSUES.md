# YouQian血压助手 V1.5 已知问题与版本说明 📋

针对当前版本（V1.5）遇到的两个核心问题，在此进行详细的记录与原理说明：

---

## 1. 血压状态始终显示“中重度高血压”及数据卡死问题

### 现象描述
* 用户历史记录中的所有测量状态均显示为红色“中重度高血压”标签。
* 在点击“保存”录入最新的正常血压（如 120/80）后，主页看板的“最后一次测量数据”卡片依然停留在以前的 `121/81` 脏数据上无法更新，且点击“趋势”、“报告”等标签页切换无响应。

### 原因排查与技术细节
1. **数据本身的临床评级**：
   * 导出的 Excel 历史记录显示，用户测得的高压主要分布在 `158 ~ 175` 之间，低压在 `105 ~ 116` 之间。
   * 根据世界卫生组织（WHO）及《中国高血压防治指南》临床判定标准：**收缩压 ≥ 160 mmHg** 或 **舒张压 ≥ 100 mmHg** 即诊断为 **2级或以上高血压（中重度高血压）**。
   * 因此，算法根据真实测量值准确评估了健康等级并给出了医疗建议，并非计算逻辑 Bug。
2. **数据卡死与UI失效根源**：
   * 本地 `localStorage` 中被误写入了一条未来时间为 **`202606-02-21 20:00`** （年份异常偏大，放大了20多万年）的极端脏数据。
   * 由于系统对记录进行降序时间排序，这条错误数据永远被排在数组的第 1 位（即最新测量数据），阻断了新录入数据的展示。
   * 此外，该异常数据导致 Chart.js 折线图重绘等环节抛出未捕获异常，阻断了后续事件绑定，造成了 Tab 切换失灵。

### 修复与优化方法（已在 V1.5 落地）
* **脏数据智能清洗**：在应用启动的 `init()` 阶段，增加年份范围过滤（仅保留合理区间 `2000-2099`），自动识别并剔除了未来的异常脏记录。
* **已有历史状态校准**：增加了数据重校准机制。如果历史缓存的等级标签与最新标准不符，会在启动时一键重算重写，消除历史缓存偏差。
* **高兼容性导出对齐**：将报告明细表格状态列的 Flex 居中布局重构为基于最经典的 `text-align: center` 和 `inline-flex` 布局，完美解决了 `html2canvas` 导出长图时状态栏偏左不对齐的引擎 Bug，并美化了徽章样式。
* **版本强制更新**：在 `index.html` 中引入 `app.js?v=18` 强制清理了 Service Worker 和浏览器的 HTTP 强缓存，确保用户直接加载最新的逻辑。

## 2. 拍照/相册图片 OCR 识别不灵敏问题

### 现象描述
* 用户使用手机拍照识别血压计屏幕读数时，偶尔无法识别、识读失败，或者读取出不合逻辑的噪点数字。

### 原因分析与技术细节
* **纯前端 OCR 的局限性**：该应用在移动端完全依靠纯前端的 `Tesseract.js` + Canvas 离线二值化处理（Bradley 自适应算法与直方图剪切双通道）实现识别。所有计算都在设备本地进行以保障隐私，不依赖任何云端高算力视觉 API。
* **物理环境依赖高**：液晶段码显示屏受反光、反差和光照影响极大。如果照片存在以下情况，识别率会明显降低：
  1. **严重反光**：日光灯或强烈反光导致数字断笔、段码反差丧失。
  2. **拍摄倾斜与抖动**：手机没有与血压计液晶屏水平平行对齐，产生了透视变形，或对焦模糊。
  3. **噪点干扰**：血压计机身上的型号信息（如“710”）或边框阴影被误判定为数字笔画。

### 建议操作指引
* **对准对焦**：将血压仪平放，手机摄像头尽量与液晶屏平行，等待相机清晰对焦后再行按键。
* **光线适宜**：避开直射强光，防止液晶屏表面反光折射。
* **备用手段**：当外部光线不佳、反光难以消除时，建议使用本软件极速优化过的手动表单或门诊多次录入模式，能提供最稳健的数据安全保障。
